学習の進め方

ネットワークは、最初からOSI参照モデルやプロトコル名を暗記しようとすると難しく感じます。まずは、身近な「家のWi-FiでWebページを見る」場面から考えると理解しやすくなります。

ゴール この教材では、「URLを入力してからWebページが表示されるまでに、どの技術がどの順番で働くのか」を説明できる状態を目指します。

おすすめの順番

  1. 家庭・職場のネットワーク全体像をつかむ
  2. Webページを見る通信の流れを理解する
  3. DNS、DHCP、ARPを役割ごとに整理する
  4. ルータ、スイッチ、アクセスポイント、ファイアウォールの違いを押さえる
  5. IPアドレス、MACアドレス、TCP、UDPを理解する
  6. 最後にOSI参照モデルへ対応させる

最初に覚える大枠

DNS

名前からIPアドレスを調べる仕組みです。例：www.example.com → 203.0.113.10

IP

ネットワーク上の住所です。ネットワークを越えて相手へ届けるために使います。

TCP

データを確実に届けるための仕組みです。Web通信でよく使われます。

HTTP

Webページを「ください」と要求し、結果を受け取るための約束です。

この教材で特に重視すること

  • 図を見ながら、通信の順番を説明できること
  • DNS、IP、TCP、HTTP、Ethernetの役割が混ざらないこと
  • OSI参照モデルを実際の通信と結びつけること
  • 基本情報技術者試験で問われやすい言い回しに慣れること

何の図か

この図は、家や職場でPCやスマホがインターネットへ接続するときの基本構成を示しています。まずは、「端末」「アクセスポイント」「スイッチ」「ルータ」「インターネット」「サーバ」の位置関係を理解します。

登場する機器や要素の役割

要素役割身近なイメージ
PC・スマホ通信の出発点です。ブラウザやアプリを使います。利用者本人

通信の流れ

  1. 利用者がPCやスマホでWebページを開こうとします。
  2. 端末は、Wi-FiやLANケーブルを通じてLANに入ります。
  3. LAN内では、アクセスポイントやスイッチが通信を中継します。
  4. インターネット上の相手に向かう通信は、ルータへ渡されます。
  5. ルータは、家庭や職場のLANからインターネット側へ通信を送ります。
  6. Webサーバから戻ってきた応答も、逆向きに端末へ戻ります。
注意 家庭用の「Wi-Fiルータ」は、名前はルータでも、実際にはルータ、スイッチ、アクセスポイント、DHCP、NAT、簡易ファイアウォールなどをまとめて持っていることが多いです。

初心者が混乱しやすいポイント

  • スイッチは、主に同じLAN内の通信を扱います。
  • ルータは、LANとインターネットなど、異なるネットワーク間の通信を扱います。
  • アクセスポイントは、Wi-Fi端末をネットワークに接続する入口です。
  • 家庭用機器では複数の機能が一体化しているため、機器名ではなく機能で考えることが重要です。

基本情報技術者試験での重要ポイント

  • スイッチ：同じLAN内で通信を中継する
  • ルータ：異なるネットワーク間で通信を中継する
  • アクセスポイント：無線LAN端末を接続する
  • ファイアウォール：通信を許可・遮断する

確認問題

  1. スイッチは、主にどの範囲の通信を中継しますか。
  2. ルータは、どのようなネットワーク同士をつなぎますか。
  3. アクセスポイントの役割は何ですか。
  4. 家庭用Wi-Fiルータが複数の機能を持つ理由を説明してください。
  5. Webサーバは、利用者PCに対して何を返しますか。
解答例を表示
  1. 同じLAN内の通信を中継します。
  2. LANとインターネットなど、異なるネットワーク同士をつなぎます。
  3. Wi-Fi端末をネットワークに接続します。
  4. 家庭では1台で通信に必要な機能をまとめると便利だからです。
  5. HTML、画像、CSS、JavaScriptなどのWebページのデータを返します。

02 Webページを見るときの通信の流れ

何の図か

この図は、ブラウザでWebページを見るとき、通信がどの順番で進むかを示しています。ネットワーク学習の中心になる図です。

sequenceDiagram
    autonumber
    participant User as 利用者
    participant PC as ブラウザ・PC
    participant DNS as DNSサーバ
    participant GW as ルータ
    participant WEB as Webサーバ

    User->>PC: URLを入力する
    PC->>DNS: ドメイン名を問い合わせる
    DNS-->>PC: WebサーバのIPアドレスを返す
    PC->>GW: Webサーバ宛ての通信を送る
    GW->>WEB: インターネット経由で中継する
    PC->>WEB: TCP接続を確立する
    PC->>WEB: HTTPリクエストを送る
    WEB-->>PC: HTTPレスポンスを返す
    PC-->>User: Webページを表示する

全体の流れ

Webページを見るときは、いきなりページのデータが返ってくるわけではありません。まず、名前からIPアドレスを調べ、通信相手を特定し、接続を準備してから、Webページを要求します。

  1. URLを入力する：利用者がブラウザにURLを入力します。
  2. DNSでIPアドレスを調べる：ドメイン名からWebサーバのIPアドレスを取得します。
  3. ルータへ通信を渡す：相手がLANの外にいる場合、端末はルータへ通信を送ります。
  4. TCP接続を確立する：Webサーバと信頼性のある通信を行う準備をします。
  5. HTTPリクエストを送る：「このページをください」と依頼します。
  6. HTTPレスポンスを受け取る：WebサーバがHTMLなどを返します。
  7. ブラウザが表示する：受け取ったデータをブラウザが画面に表示します。

DNS、IP、TCP、HTTP、Ethernetの関係

用語役割Web閲覧での位置づけたとえ
DNS名前からIPアドレスを調べる最初の住所調べ住所録
IPネットワークを越えて相手へ届ける目的地を示す住所住所
TCP通信の信頼性を管理するHTTPの前に接続を準備する配送確認
HTTPWebページを要求・応答するページをください、という依頼注文内容
Ethernet / Wi-Fi同じLAN内でデータを運ぶPCからルータへ届ける近所内の配送
具体例

家のWi-FiでWebページを見る場合、PCはまずDNSでWebサーバのIPアドレスを調べます。次に、Webサーバが家の外にいると判断し、同じLAN内の出口であるルータへデータを渡します。その後、TCPで接続を準備し、HTTPでWebページを要求します。

OSI参照モデルとの対応

OSI層関係する技術Web通信での役割
第7層 アプリケーション層HTTP、DNSWebページの要求、名前解決
第4層 トランスポート層TCP通信の信頼性を管理
第3層 ネットワーク層IP異なるネットワークへ届ける
第2層 データリンク層Ethernet、Wi-Fi、MAC同じLAN内で届ける
第1層 物理層LANケーブル、電波信号として送る

初心者が混乱しやすいポイント

  • HTTPはWebページを要求するためのルールですが、その前にDNSやTCPが関係します。
  • DNSもHTTPもアプリケーション層ですが、役割は別です。
  • IPアドレスは最終目的地を示しますが、同じLAN内ではMACアドレスも使います。
  • EthernetやWi-Fiは、同じLAN内で実際にデータを運ぶ仕組みです。

基本情報技術者試験での重要ポイント

次の順番を説明できるようにしておくと、関連問題を解きやすくなります。

  1. URLを入力する
  2. DNSでIPアドレスを調べる
  3. TCPで接続を確立する
  4. HTTPでページを要求する
  5. WebサーバがHTTPレスポンスを返す
  6. IPやEthernetにより、各区間でデータが運ばれる

確認問題

  1. Webページを見るとき、DNSは何を調べますか。
  2. HTTPリクエストは、何を依頼する通信ですか。
  3. TCPは、Web通信の中でどのような役割を持ちますか。
  4. IPアドレスとMACアドレスは、それぞれどの場面で使われますか。
  5. DNS、TCP、HTTPを通信の流れに沿って並べてください。
解答例を表示
  1. ドメイン名に対応するIPアドレスを調べます。
  2. Webページなどのデータをサーバへ要求します。
  3. データを正しく届けるために、接続、順序、再送などを管理します。
  4. IPアドレスはネットワークを越えて相手を指定するときに使い、MACアドレスは同じLAN内で機器を指定するときに使います。
  5. DNSでIPアドレスを調べる → TCPで接続する → HTTPでページを要求する、という順番です。

03 DNSの役割

何の図か

この図は、DNSがドメイン名をIPアドレスに変換する流れを示しています。DNSは、Web通信の前準備として重要です。

sequenceDiagram
    participant PC as 利用者PC
    participant Cache as キャッシュ
    participant DNS as DNSサーバ
    participant WEB as Webサーバ

    PC->>Cache: 過去に調べた結果はある？
    alt キャッシュあり
        Cache-->>PC: IPアドレスを返す
    else キャッシュなし
        PC->>DNS: www.example.com のIPアドレスは？
        DNS-->>PC: 203.0.113.10 です
    end
    PC->>WEB: IPアドレス宛てに通信する

登場する要素の役割

要素役割
ドメイン名人間が覚えやすい名前です。例：www.example.com
IPアドレスコンピュータ同士が通信するときに使う住所です。
DNSサーバドメイン名に対応するIPアドレスを返します。
キャッシュ一度調べた結果を一時的に保存します。

通信の流れ

  1. 利用者がブラウザにURLを入力します。
  2. PCは、ドメイン名だけでは通信できないため、IPアドレスを調べます。
  3. まず、ブラウザやPC内に過去の結果がないか確認します。
  4. なければDNSサーバへ問い合わせます。
  5. DNSサーバは、対応するIPアドレスを返します。
  6. PCは、そのIPアドレス宛てに通信を始めます。

初心者が混乱しやすいポイント

  • DNSサーバは、Webページ本体を返すサーバではありません。
  • DNSが返すのは、基本的にはIPアドレスです。
  • URL全体とドメイン名は同じではありません。
  • DNSの問い合わせは、キャッシュにより省略されることがあります。
整理

https://www.example.com/index.html のうち、ドメイン名は www.example.com です。https は通信方式、/index.html はサーバ内の場所を表します。

基本情報技術者試験での重要ポイント

DNSは「名前解決」と結びつけて覚えます。DNS、DHCP、ARPは混ざりやすいため、次の違いを押さえます。

仕組み役割
DNSドメイン名 → IPアドレス
DHCPIPアドレスなどを自動配布
ARPIPアドレス → MACアドレス

確認問題

  1. DNSは何を何に変換しますか。
  2. DNSサーバは、Webページ本体を返しますか。
  3. https://www.example.com/index.html のうち、ドメイン名はどの部分ですか。
  4. DNSの結果を一時的に保存する仕組みを何といいますか。
  5. DNS、DHCP、ARPの役割をそれぞれ一言で説明してください。
解答例を表示
  1. ドメイン名をIPアドレスに変換します。
  2. 返しません。基本的にはIPアドレスを返します。
  3. www.example.com です。
  4. キャッシュです。
  5. DNSは名前解決、DHCPはIPアドレスなどの自動配布、ARPはIPアドレスからMACアドレスを調べる仕組みです。

04 DHCPの役割

何の図か

この図は、PCやスマホがネットワークに接続したとき、自動的にIPアドレスなどを受け取る流れを示しています。

sequenceDiagram
    participant PC as 新しく接続した端末
    participant DHCP as DHCPサーバ
    participant RT as ルータ

    PC->>DHCP: IPアドレスを使いたいです
    DHCP-->>PC: 192.168.1.20 を使ってください
    DHCP-->>PC: サブネットマスクは 255.255.255.0 です
    DHCP-->>PC: デフォルトゲートウェイは 192.168.1.1 です
    DHCP-->>PC: DNSサーバは 192.168.1.1 です
    PC->>RT: 受け取った設定で通信開始

登場する要素の役割

要素役割
DHCPクライアントIPアドレスなどを受け取る端末です。
DHCPサーバIPアドレスなどを自動で配布します。
デフォルトゲートウェイLANの外へ出るときの出口です。通常はルータです。
DNSサーバドメイン名をIPアドレスに変換するときに使います。

通信の流れ

  1. 端末がWi-FiやLANケーブルでネットワークに接続します。
  2. 端末は、自分が使うIPアドレスをまだ持っていません。
  3. 端末は、DHCPサーバに設定情報を要求します。
  4. DHCPサーバは、空いているIPアドレスを割り当てます。
  5. DHCPサーバは、サブネットマスク、デフォルトゲートウェイ、DNSサーバも通知します。
  6. 端末は、受け取った設定で通信を始めます。

初心者が混乱しやすいポイント

  • DHCPは、ドメイン名を変換する仕組みではありません。
  • DHCPは、端末にネットワーク設定を配布する仕組みです。
  • 家庭では、Wi-FiルータがDHCPサーバとして働くことが多いです。
  • 固定IPアドレスは手動設定、DHCPは自動設定です。

基本情報技術者試験での重要ポイント

DHCPで配布される代表的な情報は、次の4つです。

  • IPアドレス
  • サブネットマスク
  • デフォルトゲートウェイ
  • DNSサーバ

確認問題

  1. DHCPは何を自動的に配布しますか。
  2. 家庭では、どの機器がDHCPサーバの役割を持つことが多いですか。
  3. デフォルトゲートウェイとは何ですか。
  4. DHCPとDNSの違いを説明してください。
  5. DHCPで配布される代表的な情報を2つ挙げてください。
解答例を表示
  1. IPアドレスなどのネットワーク設定を配布します。
  2. 家庭用Wi-Fiルータです。
  3. LANの外へ通信するときに使う出口です。
  4. DHCPはIPアドレスなどを配布し、DNSはドメイン名をIPアドレスに変換します。
  5. IPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバなどです。

05 ARPの役割

何の図か

この図は、同じLAN内でIPアドレスからMACアドレスを調べる流れを示しています。ARPは、IP通信を実際のLAN内配送につなげる仕組みです。

sequenceDiagram
    participant PC1 as PC1
    participant LAN as 同じLAN内
    participant PC2 as PC2

    PC1->>LAN: 192.168.1.30 のMACアドレスは？
    LAN->>PC2: 問い合わせが届く
    PC2-->>PC1: 私のMACアドレスは AA:BB:CC:DD:EE:FF です
    PC1->>PC2: 取得したMACアドレス宛てに送信

登場する要素の役割

要素役割
IPアドレスネットワーク上の住所です。
MACアドレスネットワーク機器ごとの識別番号です。同じLAN内で使います。
ARPIPアドレスからMACアドレスを調べる仕組みです。

通信の流れ

  1. PC1は、同じLAN内のPC2へ通信したいと考えます。
  2. PC1は、PC2のIPアドレスを知っています。
  3. しかし、同じLAN内で実際に届けるにはMACアドレスが必要です。
  4. PC1は、ARPで「このIPアドレスを持つ機器は誰ですか」と問い合わせます。
  5. PC2は、自分のMACアドレスをPC1へ返します。
  6. PC1は、取得したMACアドレス宛てにデータを送ります。

初心者が混乱しやすいポイント

  • ARPは、ドメイン名をIPアドレスに変換する仕組みではありません。
  • ARPは、IPアドレスからMACアドレスを調べます。
  • ARPは、基本的に同じLAN内で使います。
  • 遠くのWebサーバのMACアドレスを、PCが直接調べるわけではありません。
重要 インターネット上のWebサーバへ通信する場合、PCが同じLAN内で使うMACアドレスは、通常はWebサーバではなくルータのMACアドレスです。

基本情報技術者試験での重要ポイント

仕組み変換・配布するもの
DNSドメイン名 → IPアドレス
DHCP端末へIPアドレスなどを配布
ARPIPアドレス → MACアドレス

確認問題

  1. ARPは何を何に対応付けますか。
  2. MACアドレスは、主にどの範囲で使われますか。
  3. 遠くのWebサーバのMACアドレスを、PCが直接ARPで調べますか。
  4. ルータを越えると、MACアドレスの扱いはどうなりますか。
  5. DNS、DHCP、ARPの違いを説明してください。
解答例を表示
  1. IPアドレスからMACアドレスを調べます。
  2. 同じLAN内で使われます。
  3. 通常は直接調べません。次に送る相手、たとえばルータのMACアドレスを使います。
  4. ルータを越えるたびに、次の区間で使うMACアドレスが変わります。
  5. DNSは名前解決、DHCPは設定の自動配布、ARPはIPアドレスからMACアドレスを調べる仕組みです。

同じLAN内を中継"]
    FW["ファイアウォール
通信を許可・遮断"]
    RT["ルータ
異なるネットワークへ中継"]

機器ごとの役割

機器主な役割判断の目安関係しやすい層

ファイアウォール通信を許可・遮断するルール、宛先、ポート番号など複数層

通信の流れ

  1. スマホやPCは、アクセスポイントを通じてLANに入ります。
  2. LAN内では、スイッチが機器同士の通信を中継します。
  3. LANの外へ出る通信は、ルータへ向かいます。
  4. ファイアウォールは、通信を通してよいか判断します。
  5. 許可された通信は、インターネットへ送られます。

初心者が混乱しやすいポイント

  • 家庭用Wi-Fiルータは、複数の機能が1台に入っています。
  • スイッチは同じLAN内、ルータは異なるネットワーク間、と整理します。
  • ファイアウォールは単なる中継機器ではなく、通信を許可・遮断する門番です。
  • アクセスポイントは、無線端末をLANへ入れるための入口です。

基本情報技術者試験での重要ポイント

問題文で「LAN間」「異なるネットワーク」「経路制御」が出たらルータを疑います。
「同一LAN内」「MACアドレス」「フレーム」が出たらスイッチを疑います。

確認問題

  1. スイッチは主に何を見て中継先を判断しますか。
  2. ルータは主に何を見て中継先を判断しますか。
  3. アクセスポイントの役割は何ですか。
  4. ファイアウォールの役割は何ですか。
  5. 家庭用Wi-Fiルータに複数の機能が入っている例を3つ挙げてください。
解答例を表示
  1. MACアドレスです。
  2. IPアドレスです。
  3. 無線LAN端末をネットワークに接続することです。
  4. 通信を許可・遮断することです。
  5. ルータ、スイッチ、アクセスポイント、DHCP、NAT、ファイアウォールなどです。

何の図か

この図は、IPアドレスとMACアドレスの役割の違いを示しています。IPアドレスは最終的な宛先を考えるために使い、MACアドレスは同じLAN内の次の相手へ届けるために使います。

役割の違い

項目IPアドレスMACアドレス
主な役割ネットワーク上の住所機器の識別番号
主に使う範囲ネットワークを越えた通信同じLAN内
関係する層ネットワーク層データリンク層

通信の流れ

  1. PCは、WebサーバのIPアドレスを宛先として考えます。
  2. WebサーバはLANの外にいるため、PCはルータへデータを渡します。
  3. 同じLAN内でルータへ届けるために、PCはルータのMACアドレスを使います。
  4. ルータは、次のネットワークへデータを送ります。
  5. IPアドレスは、通信の最終目的地を示し続けます。
  6. MACアドレスは、区間ごとに変わります。

初心者が混乱しやすいポイント

  • IPアドレスとMACアドレスは、どちらか一方だけが使われるわけではありません。
  • IPアドレスは「最終目的地」を考えるために使います。
  • MACアドレスは「同じLAN内で次に渡す相手」を考えるために使います。
  • インターネット上の相手へ送るときでも、まず同じLAN内のルータへ渡します。

基本情報技術者試験での重要ポイント

IPアドレスは第3層、MACアドレスは第2層と結びつけます。
「ルータはIPアドレス」「スイッチはMACアドレス」と合わせて覚えると整理しやすくなります。

確認問題

  1. IPアドレスは、主に何を示しますか。
  2. MACアドレスは、主にどの範囲で使われますか。
  3. PCがインターネット上のWebサーバへ通信するとき、同じLAN内ではまず誰のMACアドレスを使いますか。
  4. IPアドレスはOSI参照モデルのどの層に関係しますか。
  5. MACアドレスはOSI参照モデルのどの層に関係しますか。
解答例を表示
  1. ネットワーク上の住所を示します。
  2. 同じLAN内で使われます。
  3. ルータのMACアドレスを使います。
  4. ネットワーク層です。
  5. データリンク層です。

08 TCPとUDPの違い

何の図か

この図は、TCPとUDPの違いを示しています。TCPは信頼性を重視し、UDPは速さや軽さを重視します。

flowchart TB
    APP["アプリケーション
Web・動画・通話・DNSなど"]
    TCP["TCP
確認しながら確実に送る"]
    UDP["UDP
確認を少なくして素早く送る"]
    WEB["Web閲覧・メール・ファイル転送"]
    REAL["DNS・動画配信・音声通話・ゲーム"]

    APP --> TCP --> WEB
    APP --> UDP --> REAL

TCPとUDPの比較

項目TCPUDP
重視すること信頼性速さ・軽さ
接続の準備行う基本的に行わない
確認応答ある基本的に少ない
再送制御ある基本的にない
代表例Web、メール、ファイル転送DNS、動画、音声通話

通信の流れ

TCPの場合

  1. 送信側と受信側が、通信の準備をします。
  2. データを順番に送ります。
  3. 受信側は、届いたことを返事します。
  4. 届かなかったデータがあれば、送信側が再送します。
  5. 受信側は、順番を整えてアプリケーションへ渡します。

UDPの場合

  1. 送信側は、必要なデータをすぐに送ります。
  2. TCPのような細かい確認は基本的に行いません。
  3. 多少の欠けがあっても、速度やリアルタイム性を優先します。

初心者が混乱しやすいポイント

  • TCPは「必ず遅い」、UDPは「必ず速い」と単純に決まるわけではありません。
  • UDPは雑な通信ではなく、リアルタイム性を重視する目的があります。
  • DNSはUDPを使う代表例としてよく出ます。
  • Web通信はTCPと結びつけて覚えると、基本情報の範囲では理解しやすいです。
補足 最近のWebでは、HTTP/3のようにUDPの上で動く仕組みもあります。ただし、基本情報の初学段階では、まず「HTTPはTCPと関係が深い」と押さえれば十分です。

基本情報技術者試験での重要ポイント

TCPとUDPは、OSI参照モデルの第4層であるトランスポート層に関係します。試験では、信頼性、確認応答、再送制御、用途の違いが問われやすいです。

確認問題

  1. TCPが重視するものは何ですか。
  2. UDPが重視するものは何ですか。
  3. TCPでは、届かなかったデータに対して何が行われますか。
  4. UDPが向いている通信の例を2つ挙げてください。
  5. TCPとUDPは、OSI参照モデルのどの層に関係しますか。
解答例を表示
  1. 信頼性です。
  2. 速さや軽さです。
  3. 再送が行われます。
  4. DNS問い合わせ、動画配信、音声通話などです。
  5. トランスポート層です。

09 OSI参照モデルと実際の通信の対応

何の図か

この図は、OSI参照モデルの各層が、Webページを見る通信のどこに対応するかを示しています。OSI参照モデルは抽象的なので、実際の通信に結びつけて覚えます。

flowchart TB
    L7["第7層 アプリケーション層
HTTP・DNS"]
    L6["第6層 プレゼンテーション層
文字コード・暗号化など"]
    L5["第5層 セッション層
通信の開始・終了の管理"]
    L4["第4層 トランスポート層

各層の役割

層主な役割代表例
第7層 アプリケーション層利用者に近い通信ルールHTTP、DNS
第6層 プレゼンテーション層データの表現形式文字コード、暗号化、圧縮
第5層 セッション層通信の開始・維持・終了セッション管理
第4層 トランスポート層アプリ間通信の信頼性や効率を管理TCP、UDP
第3層 ネットワーク層異なるネットワークへ届けるIP、ルータ
第2層 データリンク層同じLAN内で届けるEthernet、Wi-Fi、MAC、スイッチ
第1層 物理層信号を送るLANケーブル、電波

Web通信との対応

  1. ブラウザがHTTPリクエストを作ります。
  2. 必要に応じて、DNSでIPアドレスを調べます。
  3. TCPが、通信の信頼性を管理します。
  4. IPが、目的地のネットワークへ届けるための情報を付けます。
  5. EthernetやWi-Fiが、同じLAN内で次の機器へ届けます。
  6. 最後は、LANケーブルの信号や無線の電波として送られます。

初心者が混乱しやすいポイント

  • 数字が小さい層ほど、物理的な通信に近くなります。
  • 数字が大きい層ほど、利用者やアプリケーションに近くなります。
  • OSI参照モデルは、通信を理解するための考え方です。
  • 1台の機器の中で、複数の層の処理が行われます。
  • 送信側では上から下へ包み、受信側では下から上へ取り出します。

基本情報技術者試験での重要ポイント

まずは、次の5つを優先して覚えます。

  • 第7層：HTTP、DNS
  • 第4層：TCP、UDP
  • 第3層：IP、ルータ
  • 第2層：MAC、Ethernet、スイッチ
  • 第1層：LANケーブル、電波

確認問題

  1. OSI参照モデルで、HTTPやDNSが関係する層はどこですか。
  2. TCPやUDPが関係する層はどこですか。
  3. IPやルータが関係する層はどこですか。
  4. MACアドレスやスイッチが関係する層はどこですか。
  5. 数字が小さい層ほど、何に近いと考えるとよいですか。
解答例を表示
  1. 第7層 アプリケーション層です。
  2. 第4層 トランスポート層です。
  3. 第3層 ネットワーク層です。
  4. 第2層 データリンク層です。
  5. 物理的な通信に近いと考えます。

登場する要素の役割

要素役割
プライベートIPアドレス家庭や職場など、内部ネットワークで使うIPアドレスです。
グローバルIPアドレスインターネット上で一意に使われるIPアドレスです。

通信の流れ

  1. PCやスマホは、家庭内でプライベートIPアドレスを使っています。
  2. PCがインターネット上のWebサーバへ通信しようとします。
  3. ルータは、PCのプライベートIPアドレスを、自分のグローバルIPアドレスに変換します。
  4. Webサーバから見ると、通信元はルータのグローバルIPアドレスに見えます。
  5. Webサーバが応答を返すと、ルータは変換表を見て元のPCへ戻します。
  6. PCは、Webサーバからの応答を受け取ります。

プライベートIPアドレスの代表範囲

範囲よく見る例
10.0.0.0 ～ 10.255.255.255企業ネットワークなど
172.16.0.0 ～ 172.31.255.255一部の内部ネットワーク
192.168.0.0 ～ 192.168.255.255家庭用ルータでよく使う

初心者が混乱しやすいポイント

  • 家の中のIPアドレスは、インターネット全体で一意ではありません。
  • 192.168.x.x のようなアドレスは、多くの家庭で使われています。
  • 外部のWebサーバから見ると、家の中の複数端末が同じグローバルIPアドレスに見えることがあります。
  • ルータは、どの端末の通信だったかを変換表で管理します。

基本情報技術者試験での重要ポイント

  • NATは、プライベートIPアドレスとグローバルIPアドレスを変換します。
  • 複数の内部端末が、1つのグローバルIPアドレスを共有できます。
  • IPv4アドレスの不足対策として使われます。

確認問題

  1. プライベートIPアドレスは、主にどこで使われますか。
  2. グローバルIPアドレスは、どこで一意に使われますか。
  3. NATは何を何に変換しますか。
  4. 家のPCとスマホが、外部から同じIPアドレスに見えることがある理由を説明してください。
  5. NATがIPv4アドレス不足対策になる理由を説明してください。
解答例を表示
  1. 家庭や職場などの内部ネットワークで使われます。
  2. インターネット上で一意に使われます。
  3. プライベートIPアドレスとグローバルIPアドレスを変換します。
  4. ルータが複数端末の通信を1つのグローバルIPアドレスに変換するためです。
  5. 複数の端末が1つのグローバルIPアドレスを共有できるためです。

用語集

試験前の見直し用に、重要用語を短く整理します。

DNS

ドメイン名をIPアドレスに変換する仕組みです。名前解決ともいいます。

DHCP

端末にIPアドレスなどのネットワーク設定を自動で配布する仕組みです。

ARP

IPアドレスからMACアドレスを調べる仕組みです。同じLAN内で使います。

IPアドレス

ネットワーク上の住所です。ネットワークを越えた通信で使います。

MACアドレス

ネットワーク機器ごとの識別番号です。同じLAN内で使います。

TCP

信頼性を重視する通信方式です。確認応答や再送制御があります。

UDP

速さや軽さを重視する通信方式です。DNS、動画、音声通話などで使われます。

HTTP

WebブラウザとWebサーバがデータをやり取りするための通信ルールです。

HTTPS

HTTPを暗号化して安全に使うための通信方式です。

Ethernet

有線LANなどで使われる通信方式です。主にデータリンク層に関係します。

LAN

家庭や職場など、比較的狭い範囲のネットワークです。

ルータ

異なるネットワーク同士を接続する機器です。主にIPアドレスを見ます。

スイッチ

同じLAN内の機器を接続する機器です。主にMACアドレスを見ます。

アクセスポイント

スマホやノートPCなどをWi-Fiでネットワークに接続する機器です。

ファイアウォール

通信を許可したり遮断したりする仕組みです。

NAT

プライベートIPアドレスとグローバルIPアドレスを変換する仕組みです。

プライベートIPアドレス

家庭や職場などの内部ネットワークで使うIPアドレスです。

グローバルIPアドレス

インターネット上で一意に使われるIPアドレスです。

デフォルトゲートウェイ

LANの外へ通信するときの出口です。通常はルータです。

サブネットマスク

IPアドレスのネットワーク部分とホスト部分を区別するための情報です。

総まとめ問題

最後に、複数の単元をつなげて確認します。ここが説明できると、ネットワーク分野の基礎がかなり整理できています。

  1. ブラウザにURLを入力してからWebページが表示されるまでの流れを、DNS、TCP、HTTPを使って説明してください。
  2. PCがインターネット上のWebサーバへ通信するとき、IPアドレスとMACアドレスはそれぞれどのように使われますか。
  3. DNS、DHCP、ARPの違いを、変換・配布するものに注目して説明してください。
  4. ルータとスイッチの違いを、関係するOSI層とあわせて説明してください。
  5. NATが必要になる理由を、プライベートIPアドレスとグローバルIPアドレスを使って説明してください。
解答例を表示
  1. URLを入力すると、まずDNSでドメイン名に対応するIPアドレスを調べます。その後、TCPでWebサーバとの接続を確立し、HTTPリクエストでページを要求します。WebサーバはHTTPレスポンスとしてHTMLなどを返し、ブラウザが表示します。
  2. IPアドレスは、Webサーバという最終目的地を示すために使います。MACアドレスは、同じLAN内で次に渡す相手、たとえばルータへ届けるために使います。
  3. DNSはドメイン名をIPアドレスに変換します。DHCPは端末へIPアドレスなどを配布します。ARPはIPアドレスからMACアドレスを調べます。
  4. ルータは異なるネットワーク間を中継し、主にネットワーク層に関係します。スイッチは同じLAN内を中継し、主にデータリンク層に関係します。
  5. 家庭内のプライベートIPアドレスはインターネット上で一意ではありません。NATにより、ルータがプライベートIPアドレスをグローバルIPアドレスに変換することで、インターネット上のサーバと通信できます。
次に学ぶとよいテーマ サブネットマスク、CIDR、VLAN、VPN、ポート番号、HTTPSとTLSを学ぶと、基本情報技術者試験のネットワーク分野をさらに理解しやすくなります。
