学習の進め方

ネットワークは、用語を単独で覚えるよりも、「通信がどの順番で進むか」を軸にすると理解しやすくなります。まずは、家のWi-FiでWebページを見る場面から考えます。

到達目標 URLを入力してからWebページが表示されるまでに、DNS、IP、TCP、HTTP、Ethernet、ルータ、スイッチがどのように関係するかを説明できる状態を目指します。

学習の全体像

全体像家・職場の構成
次に→
Web通信DNS→TCP→HTTP
分解→

整理→
OSI対応層と役割

最初に押さえる関係

用語役割一言でいうと
DNSドメイン名をIPアドレスに変換する住所を調べる
IPネットワークを越えて宛先を示す住所
TCP確実に届くように通信を管理する配送確認
HTTPWebページを要求・取得する注文内容

よくある誤解 「DNS→IP→ルータ→HTTP→TCP」のように、全部を1本の直線として覚えると混乱しやすくなります。実際には、利用者の目的はHTTPですが、HTTPを送る前にDNSで宛先を調べ、TCPで接続を準備します。

01 家庭・職場の基本的なネットワーク構成

何の図か

PCやスマホがインターネットへ接続するときの基本構成です。各機器の位置と役割をつかみます。

基本構成：端末からWebサーバまで

PC・スマホ通信の出発点
Wi-Fi/有線→
アクセス
ポイント無線の入口
LAN内→
スイッチ同じLAN内を接続
外部宛て→
ルータ外へ出る出口
経由→
Web
サーバページを返す

登場する機器や要素の役割

要素役割身近なイメージ
PC・スマホ通信の出発点です。ブラウザやアプリを使います。利用者本人
アクセスポイントWi-Fi端末をLANへ接続します。無線の入口
スイッチ同じLAN内の機器をつなぎます。建物内の案内係
ルータLANとインターネットなど、異なるネットワークをつなぎます。外へ出る出口
WebサーバWebページのデータを返します。情報を持つ相手先

通信の流れ

  1. 利用者がPCやスマホでWebページを開こうとします。
  2. 端末は、Wi-FiやLANケーブルを通じてLANに入ります。
  3. LAN内では、アクセスポイントやスイッチが通信を中継します。
  4. インターネット上の相手に向かう通信は、ルータへ渡されます。
  5. ルータは、家庭や職場のLANからインターネット側へ通信を送ります。
  6. Webサーバから戻ってきた応答も、逆向きに端末へ戻ります。
注意 家庭用の「Wi-Fiルータ」は、実際にはルータ、スイッチ、アクセスポイント、DHCP、NAT、簡易ファイアウォールをまとめて持つことがあります。試験対策では、機器名ではなく「機能」で分けて考えます。

確認問題

  1. スイッチは、主にどの範囲の通信を中継しますか。
  2. ルータは、どのようなネットワーク同士をつなぎますか。
  3. アクセスポイントの役割は何ですか。
  4. 家庭用Wi-Fiルータに複数の機能が入っている例を3つ挙げてください。
  5. Webサーバは、利用者PCに対して何を返しますか。
解答例を表示
  1. 同じLAN内の通信を中継します。
  2. LANとインターネットなど、異なるネットワーク同士をつなぎます。
  3. Wi-Fi端末をネットワークに接続することです。
  4. ルータ、スイッチ、アクセスポイント、DHCP、NAT、ファイアウォールなどです。
  5. HTML、画像、CSS、JavaScriptなどのWebページのデータを返します。

02 Webページを見るときの通信の流れ

何の図か

ブラウザでWebページを見るとき、通信がどの順番で進むかを示しています。ネットワーク学習の中心となる章です。

Webページが表示されるまでの順番

1 URL入力
利用者がURLを入力します。
例：ブラウザに https://www.example.com を入力します。
2 DNS
PCはドメイン名に対応するIPアドレスを調べます。
「www.example.com のIPアドレスは何ですか」と問い合わせます。
3 IPで宛先判断
PCは、相手が自分のLAN内か外部かを判断します。
外部の相手なら、まずルータへ送ります。
4 Ethernet / Wi-Fi
同じLAN内では、MACアドレスを使ってルータへ届けます。
ここでEthernetやWi-Fiが関係します。
5 TCP接続
Webサーバと信頼性のある通信を始める準備をします。
HTTPを送る前の接続準備です。
6 HTTP要求
ブラウザが「このページをください」と依頼します。
これがHTTPリクエストです。
7 HTTP応答
WebサーバがHTMLなどを返します。
これがHTTPレスポンスです。ブラウザは受け取ったデータを表示します。

全体の流れ

  1. URLを入力する：利用者がブラウザにURLを入力します。
  2. DNSでIPアドレスを調べる：ドメイン名からWebサーバのIPアドレスを取得します。
  3. IPで宛先を判断する：相手がLAN内か、外部ネットワークかを判断します。
  4. 同じLAN内ではMACアドレスを使う：外部宛てなら、まずルータのMACアドレス宛てに送ります。
  5. TCP接続を確立する：Webサーバと信頼性のある通信を行う準備をします。
  6. HTTPリクエストを送る：「このページをください」と依頼します。
  7. HTTPレスポンスを受け取る：WebサーバがHTMLなどを返します。

DNS、IP、TCP、HTTP、Ethernetの関係

用語役割Web閲覧での位置づけOSI層
DNS名前からIPアドレスを調べる最初の住所調べ第7層
HTTPWebページを要求・取得するページをください、という依頼第7層
TCP通信の信頼性を管理するHTTPの前提になる接続第4層
IPネットワークを越えて届ける目的地を示す住所第3層
Ethernet / Wi-Fi同じLAN内でデータを運ぶPCからルータへ届ける第2層・第1層
混乱しやすい点 DNSもHTTPも第7層ですが、役割は違います。DNSは「相手のIPアドレスを調べる」、HTTPは「Webページを要求する」です。

試験での見方

問題文に「名前解決」があればDNS、「信頼性」「再送」「確認応答」があればTCP、「Webページの要求」があればHTTP、「同じLAN内」「MACアドレス」があればEthernetやARPを考えます。

確認問題

  1. Webページを見るとき、DNSは何を調べますか。
  2. HTTPリクエストは、何を依頼する通信ですか。
  3. TCPは、Web通信の中でどのような役割を持ちますか。
  4. IPアドレスとMACアドレスは、それぞれどの場面で使われますか。
  5. DNS、TCP、HTTPを通信の流れに沿って並べてください。
解答例を表示
  1. ドメイン名に対応するIPアドレスを調べます。
  2. Webページなどのデータをサーバへ要求します。
  3. 接続、順序、再送などを管理し、データを正しく届けやすくします。
  4. IPアドレスはネットワークを越えた宛先指定に使い、MACアドレスは同じLAN内で次の相手に届けるために使います。
  5. DNSでIPアドレスを調べる → TCPで接続する → HTTPでページを要求する、という順番です。

03 DNSの役割

何の図か

DNSがドメイン名をIPアドレスに変換する流れです。DNSは「名前解決」と結びつけて覚えます。

DNS：名前からIPアドレスを調べる

ブラウザwww.example.com を開きたい
問い合わせ→
DNS
サーバ名前解決
回答→
IPアドレス203.0.113.10
接続→
Web
サーバページを返す

通信の流れ

  1. 利用者がブラウザにURLを入力します。
  2. PCは、ドメイン名だけでは通信できないため、IPアドレスを調べます。
  3. ブラウザやPC内に過去の結果があれば、キャッシュを使います。
  4. なければDNSサーバへ問い合わせます。
  5. DNSサーバは、対応するIPアドレスを返します。
  6. PCは、そのIPアドレス宛てに通信を始めます。

初心者が混乱しやすいポイント

  • DNSサーバは、Webページ本体を返しません。
  • DNSが返すのは、基本的にはIPアドレスです。
  • URL全体とドメイン名は同じではありません。
  • DNSの問い合わせは、キャッシュによって省略されることがあります。

確認問題

  1. DNSは何を何に変換しますか。
  2. DNSサーバは、Webページ本体を返しますか。
  3. https://www.example.com/index.html のうち、ドメイン名はどの部分ですか。
  4. DNSの結果を一時的に保存する仕組みを何といいますか。
  5. DNS、DHCP、ARPの違いをそれぞれ一言で説明してください。
解答例を表示
  1. ドメイン名をIPアドレスに変換します。
  2. 返しません。基本的にはIPアドレスを返します。
  3. www.example.com です。
  4. キャッシュです。
  5. DNSは名前解決、DHCPはIPアドレスなどの自動配布、ARPはIPアドレスからMACアドレスを調べる仕組みです。

04 DHCPの役割

何の図か

PCやスマホがネットワークに接続したとき、自動的にIPアドレスなどを受け取る流れです。

DHCP：ネットワーク設定を自動で受け取る

端末
IPアドレスを使いたいです。
Wi-Fiに接続した直後など、端末はまだ自分のIPアドレスを持っていません。
DHCPサーバ
192.168.1.20 を使ってください。
あわせて、サブネットマスク、デフォルトゲートウェイ、DNSサーバも通知します。
端末
受け取った設定で通信を始めます。
家庭ではWi-FiルータがDHCPサーバを兼ねることが多いです。

DHCPで配布される代表的な情報

情報意味
IPアドレス端末が使うネットワーク上の住所です。
サブネットマスク同じネットワーク内か外部かを判断するための情報です。
デフォルトゲートウェイLANの外へ出るときの出口です。
DNSサーバ名前解決に使うサーバです。

確認問題

  1. DHCPは何を自動的に配布しますか。
  2. 家庭では、どの機器がDHCPサーバの役割を持つことが多いですか。
  3. デフォルトゲートウェイとは何ですか。
  4. DHCPとDNSの違いを説明してください。
  5. DHCPで配布される代表的な情報を2つ挙げてください。
解答例を表示
  1. IPアドレスなどのネットワーク設定を配布します。
  2. 家庭用Wi-Fiルータです。
  3. LANの外へ通信するときに使う出口です。
  4. DHCPはIPアドレスなどを配布し、DNSはドメイン名をIPアドレスに変換します。
  5. IPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバなどです。

05 ARPの役割

何の図か

同じLAN内で、IPアドレスからMACアドレスを調べる流れです。ARPは、IP通信をLAN内の実配送につなげます。

ARP：IPアドレスからMACアドレスを調べる

PC1
192.168.1.30 のMACアドレスは誰ですか。
同じLAN内へ問い合わせます。
PC2
私です。MACアドレスは AA:BB:CC:DD:EE:FF です。
該当する端末が応答します。
PC1
取得したMACアドレス宛てに送ります。
同じLAN内の配送にMACアドレスを使います。

DNS・DHCP・ARPの違い

仕組み役割覚え方
DNSドメイン名 → IPアドレス名前解決
DHCP端末へIPアドレスなどを配布設定配布
ARPIPアドレス → MACアドレスLAN内の相手確認
重要 インターネット上のWebサーバへ通信する場合、PCはWebサーバのMACアドレスを直接調べません。通常は、同じLAN内の出口であるルータのMACアドレスを使います。

確認問題

  1. ARPは何を何に対応付けますか。
  2. MACアドレスは、主にどの範囲で使われますか。
  3. 遠くのWebサーバのMACアドレスを、PCが直接ARPで調べますか。
  4. ルータを越えると、MACアドレスの扱いはどうなりますか。
  5. DNS、DHCP、ARPの違いを説明してください。
解答例を表示
  1. IPアドレスからMACアドレスを調べます。
  2. 同じLAN内で使われます。
  3. 通常は直接調べません。次に送る相手、たとえばルータのMACアドレスを使います。
  4. ルータを越えるたびに、次の区間で使うMACアドレスが変わります。
  5. DNSは名前解決、DHCPは設定の自動配布、ARPはIPアドレスからMACアドレスを調べる仕組みです。

06 ルータ・スイッチ・アクセスポイント・ファイアウォールの違い

何の図か

ネットワーク機器の役割の違いを示しています。どこで何を判断するかで整理します。

機器ごとの担当範囲

アクセスポイント

Wi-Fi端末をLANへ入れる。無線の入口。

スイッチ

同じLAN内を中継する。主にMACアドレスを見る。

ルータ

異なるネットワークへ中継する。主にIPアドレスを見る。

ファイアウォール

通信を許可・遮断する。門番の役割。

機器主な役割判断の目安関係しやすい層
アクセスポイント無線LAN端末を接続するWi-Fiでつなぐ第1〜2層
スイッチ同じLAN内を中継するMACアドレス第2層
ルータ異なるネットワークへ中継するIPアドレス第3層
ファイアウォール通信を許可・遮断する宛先、ポート番号、ルールなど複数層

試験での見分け方

  • 「異なるネットワーク」「経路制御」「IPアドレス」→ ルータ
  • 「同一LAN内」「MACアドレス」「フレーム」→ スイッチ
  • 「無線LAN」「Wi-Fi端末」→ アクセスポイント
  • 「許可」「遮断」「ポート番号」「アクセス制御」→ ファイアウォール

確認問題

  1. スイッチは主に何を見て中継先を判断しますか。
  2. ルータは主に何を見て中継先を判断しますか。
  3. アクセスポイントの役割は何ですか。
  4. ファイアウォールの役割は何ですか。
  5. スイッチとルータの違いを、OSI参照モデルと関連付けて説明してください。
解答例を表示
  1. MACアドレスです。
  2. IPアドレスです。
  3. 無線LAN端末をネットワークへ接続することです。
  4. 通信を許可・遮断することです。
  5. スイッチは主に第2層で同じLAN内を中継し、ルータは主に第3層で異なるネットワーク間を中継します。

07 IPアドレスとMACアドレスの違い

何の図か

IPアドレスは最終的な宛先を考えるために使い、MACアドレスは同じLAN内の次の相手へ届けるために使います。

IPは最終目的地、MACは次の相手

PCIP: 192.168.1.20
MAC: AA-AA
MAC宛先はルータ→
ルータIP: 192.168.1.1
MAC: BB-BB
IP宛先はWebサーバ→
Web
サーバIP: 203.0.113.10
項目IPアドレスMACアドレス
主な役割ネットワーク上の住所機器の識別番号
主に使う範囲ネットワークを越えた通信同じLAN内
関係する層第3層 ネットワーク層第2層 データリンク層
例192.168.1.20AA:BB:CC:DD:EE:FF
誤解注意 Webサーバへ通信するとき、IPの最終宛先はWebサーバです。ただし、自宅LAN内で最初に届ける相手はルータなので、同じLAN内ではルータのMACアドレスを使います。

確認問題

  1. IPアドレスは、主に何を示しますか。
  2. MACアドレスは、主にどの範囲で使われますか。
  3. PCがインターネット上のWebサーバへ通信するとき、同じLAN内ではまず誰のMACアドレスを使いますか。
  4. IPアドレスはOSI参照モデルのどの層に関係しますか。
  5. MACアドレスはOSI参照モデルのどの層に関係しますか。
解答例を表示
  1. ネットワーク上の住所を示します。
  2. 同じLAN内で使われます。
  3. ルータのMACアドレスを使います。
  4. 第3層 ネットワーク層です。
  5. 第2層 データリンク層です。

08 TCPとUDPの違い

何の図か

TCPは信頼性を重視し、UDPは速さや軽さを重視します。どちらもトランスポート層の代表です。

TCPとUDPの使い分け

TCP

確認しながら確実に送る。Web、メール、ファイル転送など。

UDP

確認を少なくして素早く送る。DNS、動画、音声通話など。

第4層

アプリケーション間の通信の性質を決める層。

項目TCPUDP
重視すること信頼性速さ・軽さ
接続の準備行う基本的に行わない
確認応答ある基本的に少ない
再送制御ある基本的にない
代表例Web、メール、ファイル転送DNS、動画、音声通話
補足 最近のWebではHTTP/3のようにUDPの上で動く仕組みもあります。ただし、基本情報の初学段階では、まず「Web通信はTCPと関係が深い」と押さえれば十分です。

確認問題

  1. TCPが重視するものは何ですか。
  2. UDPが重視するものは何ですか。
  3. TCPでは、届かなかったデータに対して何が行われますか。
  4. UDPが向いている通信の例を2つ挙げてください。
  5. TCPとUDPは、OSI参照モデルのどの層に関係しますか。
解答例を表示
  1. 信頼性です。
  2. 速さや軽さです。
  3. 再送が行われます。
  4. DNS問い合わせ、動画配信、音声通話などです。
  5. 第4層 トランスポート層です。

09 OSI参照モデルと実際の通信の対応

何の図か

OSI参照モデルの各層が、Webページを見る通信のどこに対応するかを示しています。

OSI参照モデル：上ほどアプリ、下ほど物理に近い

第7層
HTTP・DNS
利用者に近い通信ルール
第6層
文字コード・暗号化
データの表現形式
第5層
セッション管理
通信の開始・維持・終了
第4層
TCP・UDP
通信の信頼性や効率
第3層
IP・ルータ
異なるネットワークへ届ける
第2層
Ethernet・MAC・スイッチ
同じLAN内で届ける
第1層
LANケーブル・電波
物理的な信号

初心者が混乱しやすいポイント

  • 数字が小さい層ほど、ケーブルや電波など物理的な通信に近くなります。
  • 数字が大きい層ほど、ブラウザやアプリに近くなります。
  • OSI参照モデルは、通信を理解するための考え方です。
  • 送信側では上から下へ包み、受信側では下から上へ取り出します。

確認問題

  1. HTTPやDNSが関係する層はどこですか。
  2. TCPやUDPが関係する層はどこですか。
  3. IPやルータが関係する層はどこですか。
  4. MACアドレスやスイッチが関係する層はどこですか。
  5. 数字が小さい層ほど、何に近いと考えるとよいですか。
解答例を表示
  1. 第7層 アプリケーション層です。
  2. 第4層 トランスポート層です。
  3. 第3層 ネットワーク層です。
  4. 第2層 データリンク層です。
  5. 物理的な通信に近いと考えます。

10 NATとグローバルIP / プライベートIP

何の図か

家庭内のプライベートIPアドレスが、ルータでグローバルIPアドレスに変換されてインターネットへ出る流れです。

NAT：家の中のIPを外向けのIPに変換する

プライベートIPアドレスの代表範囲

範囲よく見る例
10.0.0.0 ～ 10.255.255.255企業ネットワークなど
172.16.0.0 ～ 172.31.255.255一部の内部ネットワーク
192.168.0.0 ～ 192.168.255.255家庭用ルータでよく使う

試験での重要ポイント

  • NATは、プライベートIPアドレスとグローバルIPアドレスを変換します。
  • 複数の内部端末が、1つのグローバルIPアドレスを共有できます。
  • IPv4アドレスの不足対策として使われます。

確認問題

  1. プライベートIPアドレスは、主にどこで使われますか。
  2. グローバルIPアドレスは、どこで一意に使われますか。
  3. NATは何を何に変換しますか。
  4. 家のPCとスマホが、外部から同じIPアドレスに見えることがある理由を説明してください。
  5. NATがIPv4アドレス不足対策になる理由を説明してください。
解答例を表示
  1. 家庭や職場などの内部ネットワークで使われます。
  2. インターネット上で一意に使われます。
  3. プライベートIPアドレスとグローバルIPアドレスを変換します。
  4. ルータが複数端末の通信を1つのグローバルIPアドレスに変換するためです。
  5. 複数の端末が1つのグローバルIPアドレスを共有できるためです。

11 ポート番号とHTTPS

なぜ追加したか

Web通信を理解するうえで、ポート番号とHTTPSも押さえておくと理解が深まります。基本情報技術者試験でも、代表的なポート番号や暗号化通信は出題されやすい分野です。

IPアドレスとポート番号の違い

IPアドレス

どのコンピュータへ届けるかを示します。建物の住所のようなものです。

ポート番号

そのコンピュータ内のどのサービスへ渡すかを示します。窓口番号のようなものです。

HTTPS

HTTPを暗号化して安全に使う仕組みです。代表的なポート番号は443です。

代表的なポート番号

サービスポート番号役割
HTTP80暗号化なしのWeb通信
HTTPS443暗号化されたWeb通信
DNS53名前解決
SMTP25メール送信
SSH22安全なリモート接続
理解のコツ IPアドレスは「どの端末か」、ポート番号は「その端末のどのアプリ・サービスか」を示します。

確認問題

  1. IPアドレスとポート番号の違いを説明してください。
  2. HTTPの代表的なポート番号は何番ですか。
  3. HTTPSの代表的なポート番号は何番ですか。
  4. DNSの代表的なポート番号は何番ですか。
  5. HTTPSがHTTPより安全とされる主な理由は何ですか。
解答例を表示
  1. IPアドレスは通信先の端末を示し、ポート番号はその端末内のサービスを示します。
  2. 80番です。
  3. 443番です。
  4. 53番です。
  5. 通信内容が暗号化されるためです。

総まとめ

最後に、複数の単元をつなげて確認します。ここを説明できれば、ネットワーク分野の基礎はかなり整理できています。

最重要の対応表

第7層

HTTP・DNS。Webページの要求や名前解決。

第4層

TCP・UDP。通信の信頼性や効率。

第3層

IP・ルータ。ネットワークを越えて届ける。

第2層

MAC・Ethernet・スイッチ。同じLAN内で届ける。

補助技術

DHCP、ARP、NAT。設定配布、MAC確認、IP変換。

Web通信

DNS → TCP → HTTP。IPとEthernetが下支え。

総まとめ問題

  1. ブラウザにURLを入力してからWebページが表示されるまでの流れを、DNS、TCP、HTTPを使って説明してください。
  2. PCがインターネット上のWebサーバへ通信するとき、IPアドレスとMACアドレスはそれぞれどのように使われますか。
  3. DNS、DHCP、ARPの違いを、変換・配布するものに注目して説明してください。
  4. ルータとスイッチの違いを、関係するOSI層とあわせて説明してください。
  5. NATが必要になる理由を、プライベートIPアドレスとグローバルIPアドレスを使って説明してください。
解答例を表示
  1. URLを入力すると、まずDNSでドメイン名に対応するIPアドレスを調べます。その後、TCPでWebサーバとの接続を確立し、HTTPリクエストでページを要求します。WebサーバはHTTPレスポンスとしてHTMLなどを返し、ブラウザが表示します。
  2. IPアドレスはWebサーバという最終目的地を示すために使います。MACアドレスは同じLAN内で次に渡す相手、たとえばルータへ届けるために使います。
  3. DNSはドメイン名をIPアドレスに変換します。DHCPは端末へIPアドレスなどを配布します。ARPはIPアドレスからMACアドレスを調べます。
  4. ルータは異なるネットワーク間を中継し、主に第3層に関係します。スイッチは同じLAN内を中継し、主に第2層に関係します。
  5. 家庭内のプライベートIPアドレスはインターネット上で一意ではありません。NATにより、ルータがプライベートIPアドレスをグローバルIPアドレスに変換することで、インターネット上のサーバと通信できます。

用語集

試験前の見直し用に、重要用語を短く整理します。

DNS

ドメイン名をIPアドレスに変換する仕組み。名前解決ともいいます。

DHCP

端末にIPアドレスなどのネットワーク設定を自動で配布する仕組み。

ARP

IPアドレスからMACアドレスを調べる仕組み。同じLAN内で使います。

IPアドレス

ネットワーク上の住所。ネットワークを越えた通信で使います。

MACアドレス

ネットワーク機器ごとの識別番号。同じLAN内で使います。

TCP

信頼性を重視する通信方式。確認応答や再送制御があります。

UDP

速さや軽さを重視する通信方式。DNS、動画、音声通話などで使われます。

HTTP

WebブラウザとWebサーバがデータをやり取りするための通信ルール。

HTTPS

HTTPを暗号化して安全に使うための通信方式。

Ethernet

有線LANなどで使われる通信方式。主にデータリンク層に関係します。

LAN

家庭や職場など、比較的狭い範囲のネットワーク。

ルータ

異なるネットワーク同士を接続する機器。主にIPアドレスを見ます。

スイッチ

同じLAN内の機器を接続する機器。主にMACアドレスを見ます。

アクセスポイント

スマホやノートPCなどをWi-Fiでネットワークに接続する機器。

ファイアウォール

通信を許可したり遮断したりする仕組み。

NAT

プライベートIPアドレスとグローバルIPアドレスを変換する仕組み。

ポート番号

通信先の端末内で、どのサービスに渡すかを示す番号。

デフォルトゲートウェイ

LANの外へ通信するときの出口。通常はルータです。
