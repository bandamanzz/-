学習方針

ネットワーク分野は、用語を単独で暗記するよりも、通信の流れに結びつけて理解すると安定します。基本情報技術者試験では、難しい構築手順よりも、代表的な用語・役割・違いを正しく選べることが重要です。

到達目標 URLを入力してからWebページが表示されるまでの流れを、DNS、IP、ARP、TCP、HTTP、Ethernet、ルータ、NATと結びつけて説明できる状態を目指します。

この教材で重視すること

  • 用語の意味だけでなく、どの場面で使うかを理解する
  • 似ている用語を比較して覚える
  • 図は、流れや判断軸が分かるものに絞る
  • 最後は過去問・予想問題で、選択肢の切り分けに使う
正直な前提 この教材はネットワーク分野の基礎固めとしてはかなり厚めにしています。ただし、試験合格を「絶対」と断言することはできません。仕上げには、参考書や問題演習で出題形式に慣れることが必要です。

イントラネット組織内向けの内部ネットワーク社内、庁内、校内などの「誰が使うか」で考えます。

利用対象で見る

つながり方で見る

イントラネットは何の一種か

ご指摘のとおり、イントラネットもネットワークの一種です。 ただし、LANやWANと並べるときは「同じ分類軸ではない」ことが大切です。

03 ネットワーク接続機器

この章の目的

機器の名前ではなく、どの範囲で何を判断するかで整理します。

機器主な役割判断材料関係する層
リピータ信号を増幅・再生します。物理信号第1層
ハブ受け取った信号を複数ポートへ送ります。物理信号第1層
ブリッジLAN同士を接続し、フレームを中継します。MACアドレス第2層
スイッチ同じLAN内でフレームを中継します。MACアドレス第2層
ルータ異なるネットワーク間でパケットを中継します。IPアドレス第3層
ゲートウェイ異なるプロトコルやネットワークを接続します。用途による複数層
アクセスポイント無線端末をLANに接続します。Wi-Fi接続第1〜2層
ファイアウォール通信を許可・遮断します。IP、ポート、プロトコルなど複数層
試験での見分け方 「MACアドレス」「フレーム」が出たら第2層・スイッチ、「IPアドレス」「経路制御」が出たら第3層・ルータを考えます。

補足：周辺機器と回線終端装置

機器役割理解のポイント
NIC端末をネットワークへ接続するためのインタフェースです。有線LANポートや無線LAN機能も含めて考えると分かりやすいです。
ONU光回線の信号を家庭や事業所で使える形に変換する装置です。光回線の終端装置です。
モデム回線に応じて信号を変換する装置です。ADSLやケーブル回線などで見かけます。

分類で整理する

ネットワーク機器は、どの層で何を見て判断するかで分類すると覚えやすくなります。

ネットワーク機器の大まかな分類

物理信号を扱う機器

リピータ、ハブ。主に第1層を意識します。

LAN内を中継する機器

ブリッジ、スイッチ。MACアドレスを見ます。

ネットワーク間を中継する機器

ルータ、ゲートウェイ。IPアドレスやプロトコルを見ます。

分類主な機器判断の軸
物理層中心の機器リピータ、ハブ信号そのものを扱います。
データリンク層中心の機器ブリッジ、スイッチMACアドレスを見ます。
ネットワーク層中心の機器ルータIPアドレスを見ます。
複数機能を持つ機器家庭用Wi-Fiルータルータ、スイッチ、アクセスポイント、DHCP などをまとめて持つことがあります。

流れの詳細

  1. 利用者がブラウザにURLを入力します。
  2. DNSでドメイン名に対応するIPアドレスを調べます。
  3. 宛先IPアドレスが同じLAN内か、外部ネットワークかを判断します。
  4. 外部宛てであれば、まずデフォルトゲートウェイであるルータへ送ります。
  5. 同じLAN内でルータへ届けるために、ARPでルータのMACアドレスを調べます。
  6. TCPでWebサーバとの接続を確立します。
  7. HTTPまたはHTTPSでWebページを要求します。
  8. WebサーバがHTML、CSS、画像、JavaScriptなどを返します。

    解答例を表示

  5. DNS、TCP、HTTPを通信の順番で並べてください。
    解答例を表示
    DNS → TCP → HTTPです。

05 DNS・DHCP・ARP

この章の目的

3つともIPアドレス周辺の仕組みですが、役割が異なります。試験では混同しやすいため、変換・配布・対応付けで整理します。

DNS・DHCP・ARPの役割分担

DNS

ドメイン名 → IPアドレス。名前解決。

DHCP

端末へIPアドレスなどを自動配布。

ARP

IPアドレス → MACアドレス。同じLAN内で使用。

用語説明

用語意味具体例
DNSドメイン名からIPアドレスを調べる仕組みです。www.example.com → 203.0.113.10
名前解決名前を通信に使えるIPアドレスへ変換することです。DNSの代表的な役割
DHCP端末へIPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバなどを自動配布します。Wi-Fi接続時に自動設定される
ARP同じLAN内で、IPアドレスからMACアドレスを調べる仕組みです。ルータのMACアドレスを調べる
DNSキャッシュ過去の名前解決結果を一時保存する仕組みです。同じサイトへの再アクセスを速くする
注意 DNSサーバはWebページ本体を返しません。DNSが返すのは基本的にIPアドレスです。

DNSをもう一段整理する

名前解決の基本イメージ

利用者端末www.example.com へアクセス
問い合わせ→
DNSリゾルバ名前解決を代行
結果を返す→
IPアドレス203.0.113.10 など
用語意味理解のポイント
FQDNホスト名から最上位ドメインまで含めた完全なドメイン名です。www.example.com のような形です。
DNSリゾルバ利用者の代わりにDNS問い合わせを行う仕組みです。通常はプロバイダや組織内のDNSサーバを使います。
再帰問い合わせ最終的な回答が得られるまで問い合わせを進める方式です。利用者は途中経過を意識しません。
権威DNSサーバそのドメインに関する正式な情報を持つDNSサーバです。最終的な答えを持つ側です。

分類で整理する

DNS、DHCP、ARP はどれもアドレス関連ですが、役割が違います。 変換・配布・対応付けの3つで見ると整理しやすいです。

分類仕組み役割
変換DNSドメイン名を IPアドレスへ変換します。
配布DHCPIPアドレスや DNS サーバ情報などを自動で配ります。
対応付けARP同じ LAN 内で、IPアドレスに対応する MACアドレスを調べます。

デフォルトゲートウェイとルーティング

宛先が同じネットワーク内にいない場合、端末は通常、デフォルトゲートウェイであるルータへ通信を渡します。ルータはルーティングによって、次にどこへ送るかを判断します。

同じLAN内か、外部ネットワークかで送り先が変わる

同じネットワーク

直接相手へ送ります。第2層ではMACアドレスを使います。

別ネットワーク

まずデフォルトゲートウェイであるルータへ送ります。

ルータの役割

ルーティングテーブルなどを見て、次の行き先を選びます。

関連用語

用語意味
デフォルトゲートウェイLANの外へ通信するときの出口です。通常はルータのIPアドレスです。
ルーティング宛先ネットワークへ向かう経路を選ぶことです。

07 サブネットマスク・CIDR

この章の目的

サブネットマスクは、IPアドレスのうち、どこまでがネットワーク部分で、どこからがホスト部分かを示す情報です。基本情報技術者試験では、IPアドレスの周辺知識として重要です。

サブネットマスク：ネットワーク部分とホスト部分を分ける

IPアドレス

例：192.168.1.20。端末の住所です。

サブネットマスク

例：255.255.255.0。どこまでが同じネットワークかを示します。

CIDR

例：192.168.1.0/24。「/24」は先頭24ビットがネットワーク部分という意味です。

用語説明

用語意味例
ネットワーク部分どのネットワークに属するかを示す部分です。192.168.1.0/24 の 192.168.1 に相当
ホスト部分そのネットワーク内のどの端末かを示す部分です。192.168.1.20 の 20 に相当
サブネットマスクネットワーク部分とホスト部分の境界を示します。255.255.255.0
CIDR表記ネットワーク部分のビット数を / で表します。/24、/25、/26
ネットワークアドレスネットワークそのものを示すアドレスです。192.168.1.0/24
ブロードキャストアドレス同じネットワーク内の全端末宛てに使うアドレスです。192.168.1.255/24
ホストアドレス端末に割り当てられるアドレスです。192.168.1.1〜192.168.1.254

/24 の基本例

項目値意味
ネットワーク192.168.1.0/241つのネットワークを表します。
サブネットマスク255.255.255.0先頭24ビットがネットワーク部分です。
ネットワークアドレス192.168.1.0端末には通常割り当てません。
ブロードキャストアドレス192.168.1.255端末には通常割り当てません。
利用可能なホスト192.168.1.1〜192.168.1.254端末に割り当てられる範囲です。

よく使うCIDRの目安

CIDRサブネットマスクアドレス数利用可能ホスト数の目安
/24255.255.255.0256個254台
/25255.255.255.128128個126台
/26255.255.255.19264個62台
/27255.255.255.22432個30台
/28255.255.255.24016個14台
計算の考え方 IPv4ではホスト部分が n ビットあると、アドレス数は 2^n 個です。通常、ネットワークアドレスとブロードキャストアドレスを除くため、利用可能ホスト数は 2^n - 2 と考えます。

同じネットワークかどうかの見方

/24なら、先頭3つが同じかをまず見る

同じ

192.168.1.20/24 と 192.168.1.30/24 は同じネットワークです。

違う

192.168.1.20/24 と 192.168.2.30/24 は別ネットワークです。

外部宛て

別ネットワークなら、通常はデフォルトゲートウェイへ送ります。

  1. サブネットマスクは何を示す情報ですか。
    解答例を表示

  2. CIDR表記の /24 は何を意味しますか。
    解答例を表示
    先頭24ビットがネットワーク部分という意味です。
  3. ネットワークアドレスとは何ですか。
    解答例を表示
    ネットワークそのものを示すアドレスです。
  4. ブロードキャストアドレスとは何ですか。
    解答例を表示
    同じネットワーク内の全端末宛てに使うアドレスです。
  5. 192.168.1.0/24 で利用可能なホスト数はいくつですか。
    解答例を表示
    254台です。
  6. 192.168.1.20/24 と 192.168.2.20/24 は同じネットワークですか。
    解答例を表示
    別ネットワークです。

分類で整理する

サブネットマスクや CIDR は、IPアドレスをどこまでネットワーク部として扱うかを決めるための考え方です。

分類意味例
ネットワーク部同じネットワークかどうかを判断する部分です。192.168.1.0/24 の 192.168.1 の部分

迷いやすい組合せ違い
DNSとDHCPDNSは名前解決、DHCPはIP設定の自動配布です。
DNSとARPDNSはドメイン名→IP、ARPはIP→MACです。
スイッチとルータスイッチは同一LAN内、ルータは異なるネットワーク間です。
IPアドレスとMACアドレスIPはネットワーク上の住所、MACは機器の識別番号です。
TCPとUDPTCPは信頼性、UDPは軽さ・速さです。
NATとファイアウォールNATはアドレス変換、ファイアウォールは通信の許可・遮断です。
VLANとVPNVLANはLANを論理的に分ける、VPNは離れた場所を安全につなぐ仕組みです。

総まとめ問題

  1. URLを入力してからWebページが表示されるまでの流れを説明してください。
    解答例を表示
    DNSでIPアドレスを調べ、必要に応じてARPで次の相手のMACアドレスを調べ、TCPで接続し、HTTP/HTTPSでページを要求します。
  2. DNS、DHCP、ARPの違いを説明してください。
    解答例を表示
    DNSは名前解決、DHCPはIP設定の自動配布、ARPはIPアドレスからMACアドレスを調べる仕組みです。
  3. IPアドレスとMACアドレスの違いを説明してください。
    解答例を表示
    IPアドレスはネットワーク上の住所、MACアドレスは同じLAN内で使う機器識別番号です。
  4. サブネットマスクは何のために使いますか。
    解答例を表示
    IPアドレスのどこまでがネットワーク部分かを示し、同じネットワークかどうかを判断するために使います。
  5. NAT、NAPT、IPマスカレードの関係を説明してください。
    解答例を表示
    NATはIPアドレス変換、NAPTはIPアドレスとポート番号を使った変換、IPマスカレードはNAPTとほぼ同じ意味で使われることが多い用語です。
  6. TCPとUDPの違いを説明してください。
    解答例を表示
    TCPは信頼性重視、UDPは速さ・軽さ重視です。
  7. ファイアウォール、HTTPS、VPNの役割の違いを説明してください。
    解答例を表示
    ファイアウォールは許可・遮断、HTTPSは暗号化、VPNは安全な通信路を作る仕組みです。

追加で迷いやすい比較

迷いやすい組合せ違い
LAN と イントラネットLAN は「範囲」、イントラネットは「組織内向け」という「用途・利用対象」です。
WAN と インターネットWAN は広域ネットワーク全般、インターネットはその中でも世界規模で公開されたネットワークです。
DNS と DHCPDNS は名前解決、DHCP はIPアドレスなどの自動配布です。
スイッチ と ルータスイッチは主に同一LAN内、ルータは異なるネットワーク間を中継します。

用語集

試験前に見直すための短い用語集です。

DNS

ドメイン名をIPアドレスに変換する仕組み。名前解決。

DHCP

IPアドレス、サブネットマスク、デフォルトゲートウェイなどを自動配布する仕組み。

ARP

IPアドレスからMACアドレスを調べる仕組み。同じLAN内で使います。

ICMP

IP通信の制御やエラー通知に使われるプロトコル。pingで使われます。

IPアドレス

ネットワーク上の住所。

MACアドレス

ネットワーク機器の識別番号。

サブネットマスク

IPアドレスのネットワーク部分とホスト部分を分ける情報。

CIDR

192.168.1.0/24 のようなネットワーク範囲の表記方法。

ネットワークアドレス

ネットワークそのものを示すアドレス。

ブロードキャストアドレス

同じネットワーク内の全端末宛てに使うアドレス。

NAT

プライベートIPアドレスとグローバルIPアドレスを変換する仕組み。

NAPT

IPアドレスとポート番号を使って複数端末の通信を区別しながら変換する仕組み。

IPマスカレード

NAPTとほぼ同じ意味で使われることが多い用語。

TCP

信頼性を重視する通信方式。

UDP

速さや軽さを重視する通信方式。

ポート番号

端末内のどのサービスに渡すかを示す番号。

HTTP

Webページを要求・応答するプロトコル。

HTTPS

HTTPをTLSで暗号化した通信。

SMTP

メール送信に使うプロトコル。

POP3

メールを端末へ取り込むためのプロトコル。

IMAP4

メールをサーバ上で管理しながら読むためのプロトコル。

SNMP

ネットワーク機器を管理・監視するためのプロトコル。

VLAN

物理構成とは別に、LANを論理的に分ける仕組み。

VPN

離れた拠点や端末を安全につなぐ仕組み。

PAN

個人の身近な範囲で使う近距離ネットワーク。

MAN

都市やキャンパスなど、中規模の範囲をつなぐネットワーク。

FQDN

ホスト名から最上位ドメインまで含めた完全なドメイン名。

DNSリゾルバ

利用者の代わりに名前解決を行う仕組み。

NIC

端末をネットワークへ接続するためのインタフェース。

ONU

光回線の信号を変換する終端装置。

syslog

ログを記録・転送するための仕組み。

NTP

機器の時刻を合わせる仕組み。

ループバックアドレス

自分自身を表す特別なIPアドレス。代表例は 127.0.0.1。

ゼロトラスト

内部外部を問わず、常に確認する考え方。

ユニキャスト

1台の相手へ送る通信方式。

ブロードキャスト

同じネットワーク内の全体へ送る通信方式。

マルチキャスト

特定のグループへ送る通信方式。

リンクローカルアドレス

同じリンク内だけで使う特別なアドレス。自動設定時などに使われます。

リモートアクセスVPN

個人の端末から組織へ安全に接続する VPN。

拠点間VPN

離れた拠点どうしを安全に接続する VPN。
